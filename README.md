# Byte Decoder Test

Тестовое C++/Qt решение-модуль для декодирования байтового потока в текстовые значения с последующей записью результата в выходной поток.

Каждый входной байт интерпретируется как значение одного из нескольких типов, после чего преобразуется в строку и записывается в output. Архитектура разделена на декодирование, абстракции ввода-вывода, рабочий поток обработки и обёртка над `QIODevice`.

## Задача

Модуль читает байты из источника данных, декодирует каждый байт по внутреннему формату и записывает человекочитаемое представление в приёмник данных.

Структура входных данных - один байт, который состоит из:
- старшие 2 бита — код типа;
- младшие 6 бит — полезное значение.

Поддерживаемые типы:
- `00` — unsigned value;
- `01` — signed 6-bit value;
- `10` — латинская буква в диапазоне `a..z`;
- `11` — зарезервированный/невалидный тип.

Каждое декодированное значение записывается как отдельная строка. Невалидные значения пропускаются и в выход не попадают. Это явно проверяется тестами.

Примеры:
- unsigned: `00 101010` → `42`
- char: `10 000000` → `a`
- signed: `01 111111` → `-1`
- signed: `01 100000` → `-32`

## Архитектура:
### Интерфейсы ISink и ISource
Абстракции классов ввода-вывода. Позволяют тестировать `ConverterWorker`, не зависев от конкретной реализации чтения-записи (`MockSink` + `MockSource`).
### ByteDecoder
Отвечает за декодирование одного байта в строку по описанным в задаче типам. При невалидном результате главная функция decode возвращает `std::nullopt`.

Задачи:
- Определять тип по 2 старшим битам
- Извлекать данные из 6 младших бит
- Преобразовывать данные в строковое представление
- Отдавать `std::nullopt` при невалидных входных даннных

### ConverterWorker
Связывает `source`, `decoder`, `sink`.

Задачи:
- Открывать `source`, `sink`
- Вызывать `read()` для `source`
- Передавать полученный байт в decoder, обрабатывать строковый результат
- Вызывать `write()` для `sink` с строковым результатом, добавление разделителя (`\n` зашито в коде)
- При завершении рабочего цикла вызывать `flush()` для sink и закрывать ресуры (реализация `close()` для `QIODevice<...>`)

### Converter
Класс, предоставляющий апи (старт, стоп, проверка запуска) для работы с ConverterWorker.

## Тесты
### ByteDecoder
- Корректное декодирование unsigned-значений
- Корректное декодирование signed 6-bit значений
- Корректное декодирование char значений
- Reserved type даёт невалидный результат
- Недопустимые символьные значения не декодируются

### ConverterWorker
- `Worker` открывает `source` и `sink`
- Читает последовательность байтов
- Записывает только валидные значения
- Добавляет `\n` после каждого значения
- Пропускает невалидные байты
- Завершает работу через `flush()` и `close()`

### QIODevices
- Чтение байтов из `QBuffer`/`QIODevice`
- Корректная работа `open()`
- Корректный `eos()`
- Корректная обработка пустого устройства
- Успешная запись строки
- Корректный `flush`
- Корректное закрытие устройства
- Накопление текста в выходном буфере

### Converter
- `Converter` правильно связывает `QIODevice`-уровень с `ConverterWorker`
- Входные байты из Qt-буфера реально преобразуются в итоговую строку
- Результат соответствует формату output worker-а

Проект собирается через CMake. В конфигурации используются Qt и GoogleTest.

- CMake
- C++-совместимый компилятор
- Qt 6 (модуль Core)
- GoogleTest
